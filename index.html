About

The shortcapri project is about developing a simple, demonstrative example for the CAPRI market model. The model has been also used for an ongoing research investigating the impacts of different TRQ representations in applied equilibrium models. A conference paper featuring the current state is to be presented on the 135th EAAE Seminar: Himics, Britz: "Trade diversion effects of Preferential Trade Agreements under Tariff Rate Quota regimes"

I aimed at creating the simplest structure possible, making as many simplifications as I could: the current structure has only two products, three regions, no processing and no feeds, etc. But the core of the CAPRI market model is well represented:

  • similar nested set definitions as in CAPRI
  • the same variable/equation/parameter... names
  • the same Generalized Leontief demand system
  • the same type of supply functions (although without the fudging for negative values)
  • the same two stage Armington approach for trade
  • similar linkages between market/Armington/consumer/producer prices
  • the same approach for calibrating the price elasticities of human demand
  • the same approach for calibrating the price elasticities of supply

Extra features

The model is formulated as an MCP, but an equivalent NLP formulation is also given in the code. Remember, we solve a (squared) system of equations using the complementarity conditions. The MCP formulation shows explicitely the equation-variable pairs. The NLP formulation is the one used in CAPRI.

The code contains lengthy routines for checking the calibration. This might look unnecessary at the first glance but I think it's useful to emphasize which parameters are calibrated and how...

Installation

Just download the files by clicking on the zip folder above. You can also browse the code online by clicking on the blue 'Wiew on Github' button above. The main file is market_mcp.gms; that's the one you want to run.

[Those of you who are familiar with git can clone or fork the repository].

The content of the package

The main files are in the main folder. The different sub-modules (e.g. data preparation, calibration) are sitting in the folder "include". In the folder "R" you can find some R routines for reporting purposes.

The main files:

  • market_mcp.gms: the basic model
  • market_trq.gms: an exteded version with TRQs
  • market_trq_onec: the TRQ version with only one commodity
  • market_trq_one_SA...: sensitivity analyses on different aspects, e.g. Armington elasticities, TRQ rents

Why?

I used to analyse hypothetical scenarios with this and similar small models. It can be useful to get an idea what you can expect from CAPRI after implementing a real scenario there. I publish it in the hope that others might find it useful too.
